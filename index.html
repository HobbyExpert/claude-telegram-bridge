Claude Telegram Bridge

Control Claude Code on this Mac from Telegram. Multiple named sessions run concurrently, headless, auto-start on login.

Architecture

Commands

Basics
any message Runs as a Claude Code task in the active session with streaming output
photo / file Send an image or document to include in the task
/status List all sessions (active marker, running state, elapsed time) + system claude processes
/stop [name] Terminate the running task in the named session (defaults to active)
Sessions
/sessions Browse live sessions + recent JSONL history; tap to switch or resume
/switch [name] Switch the active session. No arg → inline keyboard picker
/spawn <name> [path] Create a new named session with its own cwd and become active
/kill [name] Terminate and remove a session (defaults to active)
/new Clear the active session's context (next message starts fresh)

/context Toggle auto-continue for the active session (on by default)
Settings

/turns <N> Set max agentic turns for the active session (default 10)

/id Show your Telegram user ID

How It Works

Multiple concurrent sessions
Spawn named sessions (/spawn akeneo /Users/santibm/Sites/akeneo) each with their own cwd, model, budget, and conversation history. Typed messages route to the active session. When multiple sessions exist, responses are prefixed with [name ▶] so you always know which one spoke.
Resuming a historical session
Tapping a 📜 entry in /sessions reads the JSONL to recover the exact cwd where it was recorded, sets it on the active slot, and arms auto-continue — so your next message lands in the right project with the right context.
Streaming responses
Messages edit in real time as Claude works. Tool calls show live (🔧 Bash: ls -la). Status reactions track progress: 👀 → 🤔 → 👨‍💻 → 🔍 → 👍
Dangerous command blocking
Prompts mentioning rm -rf, force push, drop table, etc. are intercepted. You get a confirm/cancel inline keyboard before execution.
Photo & file analysis
Send a screenshot, PDF, or code file directly in Telegram. It gets saved locally and passed to Claude for analysis (max 10MB).

Response UX
Done Streaming responses (edit-in-place) telclaude
Instead of waiting for Claude to finish, edit the Telegram message in real time as output streams in. Use --output-format stream-json and editMessageText with a debounced 1.5s interval. When content exceeds 4000 chars, finalize the current message and roll over to a new one.
Done Status reactions as progress indicators telclaude
Use Telegram emoji reactions on the bot's own message to show execution state — no chat clutter. Stages: 👀 queued → 🤔 thinking → 👨‍💻 coding → 🔍 searching → ✅ done → 💔 error. Parse stream-json tool events to determine which stage to show.
Done Natural boundary message splitting telclaude
When splitting long messages, search the last 20% of the limit for the best breakpoint: paragraph break > line break > sentence end > word boundary > hard cut. Prevents splitting mid-word or mid-code-block.
Done Reaction acknowledgment instead of reply claudecode-telegram
When a message is received, add a ✓ reaction to the user's message instead of sending "Working..." as a separate message. Cleaner UX, no chat clutter. The "Working..." text becomes the streaming message that gets edited in place.
Session Management
Done Multi-session registry with named slots new
Refactored the single State class into a GlobalState holding a dict of named SessionSlots. Each slot owns its own subprocess, cwd, model, budget, and conversation history. Commands: /spawn, /switch, /kill. Output is prefixed with [name ▶] when more than one session exists so messages from different sessions don't blur together.
Done Session picker with inline keyboard claudecode-telegram
/sessions shows an inline keyboard: live sessions at the top with status dots (🟢 running, ⚫ idle) and elapsed time, then recent JSONL history from ~/.claude/projects/. Tap a live one to switch active; tap a 📜 history entry to load it into the active slot.
Done Historical session resume with cwd recovery new
Tapping a JSONL history entry reads the first few lines to extract the cwd the session was originally recorded in, and sets it on the active slot. Without this, claude -r <id> ran from the wrong directory and exited silently with is_error=true and zero turns.
Done Bot command menu for / autocomplete new
Registered a full BotCommand list via bot.set_my_commands in _post_init. Typing / on Telegram now surfaces every supported command with a short description — no need to remember the exact name.
Done Session idle timeout telclaude
Auto-close session tracking after configurable idle period (default 60 min). Clear last_session_id so stale sessions aren't accidentally continued. Configurable via SESSION_IDLE_TIMEOUT env var.
Input Capabilities
Done Photo, document & file handling telclaude
Accept images, screenshots, and documents sent via Telegram. Download the file via Bot API, save to a temp path, and include it in the Claude prompt: "Analyze this image: /tmp/photo.jpg". Supports PNG, JPG, PDF, and text files. Auto-cleanup after task completes.
Done Voice message transcription telclaude
Send a voice note or audio file; the bridge downloads it, POSTs to OpenAI Whisper (whisper-1), echoes the transcript with a 👂 prefix, and forwards the text to Claude in the active session. 25MB limit. Requires OPENAI_API_KEY in .env.
Safety & Control
Done Command blocking for dangerous operations claudecode-telegram
Block messages that could trigger destructive actions through headless mode: rm -rf, git push --force, drop table, etc. Return a warning and require explicit confirmation via inline keyboard before proceeding.
Done Rate limiting telclaude
Sliding window rate limiter: RATE_LIMIT_PER_MIN (default 10) and RATE_LIMIT_PER_HOUR (default 60). Prevents accidental runaway costs from rapid messages. In-memory timestamp deque, checked before spawning each task.
Done Stale task auto-cleanup claudecode-telegram
Background watchdog sweeps every minute across all sessions. If a slot's subprocess has been running longer than TASK_TIMEOUT * 2, it's killed, its stream task cancelled, and the chat notified — so zombie processes can't block new tasks on that slot.

/projects command shows an inline keyboard with all directories under ~/Sites/. Tap one to switch CWD instantly. Much faster than typing /cwd ~/Sites/enterprise each time. Show current project with a checkmark.
